# AI赋能校园学习互助与课程资源推荐平台报告

## 一、需求分析

平台面向高校课程学习场景，解决课程资料分散、重复问题多、互助过程缺少记录、优质资源难沉淀等问题。系统角色包括管理员、教师/助教、学生。管理员负责用户、审计与系统管理；教师/助教负责资源发布、资源审核和学习数据查看；学生负责资源浏览收藏、发布求助、认领互助、评价与积分查看。

## 二、概要设计

系统采用 Spring Boot 3.x + Spring MVC + Thymeleaf + MyBatis 分层架构：

- Controller：接收页面请求，完成参数绑定和视图跳转。
- Service：封装业务规则、事务、缓存、AI 服务调用。
- Mapper：使用 MyBatis 注解 SQL 完成数据访问。
- AOP：通过 `@AuditLog` 记录关键操作、耗时和状态。
- Security：通过 Spring Security 实现表单登录、BCrypt 密码和 RBAC。

## 三、数据库设计

数据库不少于 10 张表，包括：`users`、`role`、`user_role`、`course`、`tag`、`resource`、`resource_tag`、`favorite`、`help_request`、`help_comment`、`point_record`、`ai_call_log`、`operation_log`。其中资源和求助表包含审核状态、状态流转、浏览量、积分等核心字段。资源与标签通过 `resource_tag` 建立多对多关系，支持课程资源按知识点标签筛选。求助评论表用于保存问题补充说明与讨论记录。积分明细、AI 调用日志与操作审计日志可用于追踪业务结果。

## 四、核心模块实现

用户模块实现注册、登录、退出、密码加密、个人资料维护和角色权限控制，管理员后台支持用户状态审核/禁用和角色分配。资源模块支持教师直接发布公开资源、学生上传后待审核、资源关键词检索、课程筛选、类型筛选、标签筛选、排序、分页、收藏和浏览量统计。求助模块支持发布、认领、评论/补充说明、提交解答、确认完成五个核心步骤，确认完成时使用事务同时处理发布者扣分、帮助者加分与积分流水。

## 五、AI 功能设计

AI 模块通过独立 `AiService` 实现，没有把 AI 调用写在 Controller 中。当前支持智能推荐、资源摘要、内容审核、求助 AI 辅助解答四项功能。系统默认使用 mock 模式，便于课堂无网络或无 Key 时演示；当配置 `AI_MOCK_ENABLED=false`、`AI_BASE_URL`、`AI_MODEL`、`AI_API_KEY` 后，可调用 OpenAI-compatible 的真实大模型 `chat/completions` 接口。每次调用都会写入 `ai_call_log`，包括功能类型、发起用户、请求摘要、响应摘要、耗时、状态和创建时间。

求助 AI 辅助解答用于“长期无人认领或暂时搁置”的问题。系统会将求助标题、描述、知识点标签、已有评论和平台已审核资源作为上下文发送给模型，提示模型优先基于平台资料给出问题判断、解决步骤、参考资源和下一步建议，避免脱离课程资料随意生成。

## 六、测试结果

已执行 Maven 编译测试：

```text
mvn -s settings-local.xml test
BUILD SUCCESS
```

主要人工演示流程包括：登录、管理员审核/禁用用户、管理员分配角色、资源列表按课程/类型/标签筛选和排序、学生上传资源、教师/管理员审核、发布求助、认领求助、提交解答、确认完成、查看积分明细、执行 AI 推荐/摘要/审核、查看 AI 与操作审计日志。

本次本地验证启动了 `http://localhost:8083` 演示实例，并通过脚本完成 25 项端到端检查：未登录跳转、管理员/学生权限隔离、待审和禁用账号拦截、资源筛选与分页、个人资料维护、学生上传资源、教师审核资源、资源详情与收藏、求助发布/认领/评论/解答/确认完成、AI 推荐/摘要/审核、后台用户治理、AI 日志和操作审计日志查看。全部通过，`FAILED_COUNT=0`。

## 七、遇到的问题与解决方案

Maven 默认仓库路径无写权限，导致依赖无法下载。解决方式是在项目中增加 `settings-local.xml`，将 Maven 本地仓库指定到项目 `.m2/repository`。AI API 在课堂演示环境中不一定具备外网和 Key，因此默认提供 mock 模式，同时保留 provider/base-url/api-key/timeout 等配置入口。
